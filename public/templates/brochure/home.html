About

This site was developed as a technical assessment for Mukuru. The following sections provides a quick overview of this project.

 

Frameworks

- Frameworks used to build this project -

Zend Framework 2.3


The most popular framework for modern, high-performing PHP applications.

jQuery 1.11.1


jQuery is a fast, small, and feature-rich JavaScript library. It makes things like HTML document traversal and manipulation, event handling, animation, and Ajax much simpler with an easy-to-use API that works across a multitude of browsers.

Bootstrap 3


Bootstrap is the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web.

Doctrine 2


The Doctrine Project is the home of a selected set of PHP libraries primarily focused on providing persistence services and related functionality. It's prize projects are a Object Relational Mapper and the Database Abstraction Layer it is built on top of.

 

Design Patterns

- Design patterns used in this project -

Registry pattern


The Registry pattern was used for universal access to the Zend ServiceManager, Zend Session and authenticated user data.

Repository pattern


The Repository pattern was used as a separation layer between the REST API and Service layer of the application and to keep our controllers as thin as possible.

Delegation pattern


The Delegation pattern was used for initial exchange order calculations, additional discount calculation on entity population and email notification execution based on currency rules.

Service Locator pattern


The Service Locator pattern was used for all interactions with services by the Repository layer.

 

Separation of concerns

- Principles and patterns used to achieve separation of concerns in this project -

Zend MVC


The MVC pattern was used to separate controller logic from the presentation layer.

Separate web service


A REST API was used for further separation from the presentation layer, enabling the usage of different presentation layers in the future as well as exposing functionality to external consumers.

Doctrine 2 ORM


The Doctrine 2 ORM framework was used as a database extraction layer, providing for easier switching of database technologies, a significant reduction in code and database structure creation as well as initial data population.
